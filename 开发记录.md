
## 基于 LLM 的 Markdown 文本批量翻译

### 一、项目简介

本项目旨在对大段 Markdown 文本进行自动化的分段翻译，并在翻译后依然保留原有的 Markdown 层级及格式。整体流程包括：
- 读取并解析 Markdown 文件
- 根据指定的规则进行分段（例如按字数或自然段为单位分段）
- 调用大语言模型（LLM）进行翻译
- 将翻译结果拼接为完整的 Markdown 文档，并保留原文的层级结构和格式

### 二、技术选型

- **语言**：Python（版本 3.7+ 推荐）
- **依赖库**：
  - openai（或自定义封装的 `from openai import OpenAI`）
  - markdown、markdown-it-py 或类似库（可根据需要进行文本解析和处理）
  - 其他可选的文本处理工具，如 `re`、`textwrap` 等

> 注意：本项目中使用的 `openai` 包与官方的 OpenAI 包接口略有差异，实际是对 Fireworks.ai 等服务的自定义封装；也可以切换至其他 LLM 提供商（如 Claude 3.5 Sonnet），在下文中会提到如何进行配置。

### 三、核心思路与代码结构

#### 1. 读取 Markdown 文件
- 通过 Python 的文件 I/O 将原始内容读取为字符串 `raw_text`。

#### 2. 分段逻辑
- 按约 5000 字（或自定义大小）为标准进行切分，尽量在自然段边界进行分割，以避免上下文断裂。
- 如果需要更精细地保持 Markdown 结构，可以先解析成段落/标题/列表等结构，再在段落层面上聚合形成子块。

#### 3. 翻译与调用 LLM
- 对每个分段分别调用 LLM 进行翻译。
- 调用时需要在提示词或上下文中强调需要保留 Markdown 结构、语法等。
- 可以使用流式（stream）或非流式模式获取翻译结果。
- 通过独立的 `llm_client.py` 模块，可以灵活地切换不同的 LLM 提供商与模型（如 Fireworks DeepSeek-R1、Claude 3.5 Sonnet 等）。

#### 4. 结果合并及格式保持
- 将翻译后的各段依照原始顺序拼接，生成最终的 Markdown 文本。
- 若需要在提示中保留代码块、标题层级等，可以在分段时给每个块添加相应标识，并在合并时根据标识正确插入。

### 五、示例代码片段

以下是简化的示例，演示如何使用自定义封装的 `openai`（Fireworks.ai 版）进行流式调用（stream），并打印输出。

#### `translator.py`

```python
# translator.py

from openai import OpenAI

api_key = "YOUR_API_KEY"
api_base = "https://api.fireworks.ai/inference/v1"
client = OpenAI(api_key=api_key, base_url=api_base)

def openai_llm_stream(prompt: str):
    """
    对给定的 prompt 进行翻译或处理，返回生成结果。
    可以根据需要增加 prompt 中对保持 Markdown 结构的说明。
    """

    completion = client.chat.completions.create(
        model="accounts/fireworks/models/deepseek-r1",
        stream=True,
        temperature=0.6,
        messages=[
            {"role": "system", "content": ""},
            {"role": "user", "content": prompt}
        ]
    )

    translated_text = []
    for chunk in completion:
        delta = chunk.choices[0].delta.content
        if delta:
            translated_text.append(delta)
    return "".join(translated_text)
```

#### `main.py`

在使用该函数时，需要先将你的文本分段，并在 prompt 中明确告知模型：需要保留段落、标题、列表、代码块等 Markdown 格式。

```python
# main.py

from chunker import split_text_into_segments
from translator import openai_llm_stream

def main():
    # 1. 读取 Markdown 文件
    with open("input.md", "r", encoding="utf-8") as f:
        raw_text = f.read()

    # 2. 分段
    segments = split_text_into_segments(raw_text, max_length=5000)

    # 3. 翻译 + 合并结果
    translated_segments = []
    for seg in segments:
        # 在提示词中可以加入保留 Markdown 结构的要求
        prompt = (
            "以下内容是一段 Markdown 文本，请保持原有的 Markdown "
            "结构、代码块格式、标题层级，仅翻译文字部分：\n\n" + seg
        )
        result = openai_llm_stream(prompt)
        translated_segments.append(result)

    final_text = "\n".join(translated_segments)

    # 4. 写入输出文件
    with open("output.md", "w", encoding="utf-8") as f:
        f.write(final_text)

if __name__ == "__main__":
    main()
```

### 六、如何使用

#### 1. 安装依赖
在项目根目录下执行：

```bash
pip install -r requirements.txt
```

#### 2. 配置 API Key
- **方式一**：在 `translator.py` 中填写 Fireworks.ai 等服务的 API key 和 base URL
- **方式二**：使用环境变量（推荐），例如在 `.env` 文件中写入：

```plaintext
FIREWORKS_API_KEY=your_api_key_here
FIREWORKS_API_BASE=https://api.fireworks.ai/inference/v1
```

#### 3. 运行脚本

```bash
python main.py input.md -o output.md
```

可选参数：
- `-o, --output`: 指定输出文件路径（默认 `output.md`）
- `-s, --chunk-size`: 指定分段大小（默认 5000 字符）
- `-k, --api-key`: 直接指定 API 密钥
- `-b, --api-base`: 直接指定 API 基础 URL
- `-l, --log-dir`: 指定自定义日志目录（默认：`logs/`）

#### 4. 查看结果
- 翻译完成后，会在指定输出文件或默认的 `output.md` 中保留原有的 Markdown 结构。
- 日志会记录在 `logs/` 下（或自定义目录），包含每段原文、译文、时间戳及其他元信息。

#### 使用示例
- 翻译 `example.md` 并输出到默认文件

```bash
python main.py articles/example.md
```

- 翻译指定文件并自定义输出文件名

```bash
python main.py articles/example.md -o articles/example_zh.md
```

- 指定自定义日志目录

```bash
python main.py input.md -o output.md -l custom_logs
```
